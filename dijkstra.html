Introduction

L'algorithme de Dijkstra (1959) est un algorithme glouton. Dans un graphe orienté ou non, dont les arêtes ont des poids strictement positifs, il permet de déterminer le chemin le plus court entre un sommet source et tous les autres sommets du graphe.

Le Fonctionnement

1. Initialisation

On prépare le terrain :

  • Fixer la distance à la source à $0$ : $d[S] = 0$.
  • Fixer la distance à tous les autres sommets à $+\infty$.
  • Marquer tous les sommets comme non visités.

2. Itérations

Tant qu'il reste des sommets non visités :

  1. Choix optimal : Sélectionner le sommet $u$ non visité ayant la plus petite distance $d[u]$.
  2. Visite : Marquer $u$ comme visité.
  3. Relâchement : Pour chaque voisin $v$ de $u$, tester si passer par $u$ est plus court :
si d[u] + poids(u, v) < d[v] alors
  d[v] = d[u] + poids(u, v)

Structures & Complexité

Le choix de la structure de données pour stocker les distances change radicalement l'efficacité de l'algorithme :

Tableau Classique

On cherche le minimum en parcourant tout le tableau à chaque tour.

Complexité : $O(S^2)$

File de Priorité (Tas)

L'extraction du minimum est instantanée, la mise à jour coûte $O(\log S)$.

Complexité : $O((A + S) \log S)$

Limite : Poids Négatifs

Dijkstra échoue avec des arêtes négatives car une fois qu'un sommet est marqué "visité", l'algorithme considère que son chemin le plus court est définitif. Une arête négative située plus loin pourrait pourtant diminuer ce coût. Dans ce cas, privilégiez Bellman-Ford ou Floyd-Warshall.
